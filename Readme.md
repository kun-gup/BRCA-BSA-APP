# Migration History (legacy)

These files (001-014) are the actual live-debugging history of this
project — including a few patch/fix files (005, 011, 012) written to
correct mistakes discovered while testing against a real database.

**For a NEW Supabase project, don't replay these.** Use the consolidated
files instead:
```
supabase/schema.sql    (run first)
supabase/policies.sql  (run second)
supabase/seed.sql      (run third)
```

This folder is kept only as a historical record of how the schema
evolved, in case anything here needs to be referenced later.
